# SQL Server Database Operations Project

This project demonstrates a comprehensive set of SQL Server skills, combining database design, reporting, administrative tasks, and advanced SQL routines into a single structured solution.

## Overview

The project was built to simulate real-world database operations and problem-solving. It includes creating and managing relational tables, writing complex queries, implementing triggers, handling errors, and performing administrative tasks such as backup and restore.

The goal was not just to write SQL, but to show a full understanding of how databases are built, maintained, and optimized in practice.

## Key Features

### Database Design & Data Management
- Created structured relational tables (Customer, Product, Category, Purchase, etc.)
- Defined primary keys, identity columns, and multiple indexes
- Inserted and managed realistic sample data
- Designed views for simplified data access

### Reporting & Querying
- Aggregated customer purchases and totals
- Counted customers by city
- Linked products to sales and categories
- Sorted and filtered multi-table query outputs
- Generated reports based on purchase behavior

### Advanced SQL Logic
- IF / ELSE / ELSE IF conditional routines
- CASE statements with multiple conditions
- Use of variables within SQL scripts
- Subqueries and Common Table Expressions (CTEs)
- Temporary tables for intermediate processing
- UNION operations and data transformations

### Triggers & Automation
- Audit trigger capturing INSERT, UPDATE, DELETE activity
- Cascading delete trigger to maintain data consistency
- Automated updates between related tables

### Error Handling & Logging
- Custom error logging system using @@ERROR
- Stored procedures with built-in validation and logging
- Captured useful debugging and execution information

### Performance & Optimization
- Created and tested indexes
- Compared execution times before and after indexing
- Demonstrated performance improvements in query execution

### Administrative Tasks
- Displayed active SQL Server logins
- Terminated active connections
- Created users and assigned roles
- Executed database backup and restore operations

### Data Quality & Integrity
- Prevented duplicate records using constraints
- Identified duplicate data using GROUP BY and HAVING
- Built routines to isolate and analyze duplicate entries


## Technologies Used

- Microsoft SQL Server
- T-SQL (Transact-SQL)
- SQL Server Management Studio (SSMS)
